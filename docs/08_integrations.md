# Интеграции

## Назначение документа
Цель - зафиксировать внешние системы, способы взаимодействия, назначение каждой интеграции и особенности обработки ошибок.

## Общая схема интеграций

Система построена на базе n8n, который выступает оркестратором процесса.

n8n полчает входящие документы, извлекает данные, обращается к внешним сервисам для проверок, получает данные и создает документ "Поступление товаров/услуг"

Документы из Диадок не подключаются напрямую через API. Они направяются на почту и обрабатываются системой как почтовые вложения.

## Диаграмма интреграций

## Участники интеграций
| Система | Роль в процессе |
| --- | --- | 
| n8n | Центральная система автоматизации|
| СБИС | Источник входящих документов |
| Диадок | Источник входящих документов |
| Почта | Канал получения документов и отправки уведомлений гл. бухгалтеру |
| OCR / AI API  | Сервис распознования PDF и изображений |
| DaData | Сервис проверки данных контрагента в реестре ФНС |
| 1C Fresh | Учетная система клиента |
| n8n Data Tables | Внутреннее хранилище правил поставщиков и счетов учета |
| n8n Webhooks | Механизм для реализации админстративного интерфейса |

## Таблица интеграций

| Система | Тип интеграции | Назначение | направление |
| --- | --- | --- | --- |
| СБИС | HTTP API | Получение входящих документов | CБИС --> n8n  |
| Диадок | Перенаправление на почту | Получение входящих документов | Диадок --> Почта --> n8n  |
| Почта | IMAP | Получение входящих документов | Почта --> n8n  |
| Почта | SMTP | Отправка уведомлений главному бухгалтеру | n8n  --> Почта |
| OCR / AI API | HTTP API | Распознавание сканов и извлечение данных | n8n  --> OCR / AI |
| DaData | HTTP API | Проверка контрагентов по ИНН в реестре ФНС | n8n  --> DaData |
| 1C Fresh | OData | Поиск контрагентов, договоров, документов, номенклатуры | n8n --> 1С |
| 1C Fresh | OData | Cоздание итогового документа "Поступление товаров/услуг" | n8n --> 1С |
| n8n Data Tables | Внутреннее  хранилище | Хранение правил поставщиков | n8n <--> Data Tables |
| n8n Webhooks | HTTP endpoint | Административный интерфейс | Пользователь --> n8n |

## Передаваемые данные 
### Основыные данные входящего документа
| Данные | Примеры полей | 
| --- | --- | 
| Реквизиты документа | номер, дата, тип, источник | 
| Поставщик | Наименование, ИНН, КПП | 
| Покупатель | Наименование, ИНН, КПП | 
| Договор | Номер, дата | 
| Суммы | Сумма без НДС, ставка НДС, сумма НДС, итоговая сумма | 
| Табличная часть | Наименование позиции, количество, единица измерения, цена, сумма | 
| Дополнительные реквизиты | Строки 5a и 5б | 
| Технические данные | тип файла, результат распознования, статус обработки | 

### Данные передаваемые в 1С 
| Данные | Назначение | 
| --- | --- | 
| Данные поставщика | Поиск или сопостовление контрагента | 
| Данные организации клиента | Определение базы и организации в 1С | 
| Данные договора | Привязка документа к договору | 
| Суммы и НДС | Заполнение итогов документа проверка расчетов | 
| Счет учета | Заполенине учетных счетов по правилам поставщика |
