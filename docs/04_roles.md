# Роли и участники
## Назначение документа

Цель - зафиксировать, кто участвует в процессе, какие действия выполняет каждый участник и какие системы используются. 

## Классификация участников

* Пользователи системы
* Внешние участники процесса
* Систенмые участники
* Внешние информационные системы

## Пользователи системы

| Роль	| Описание | Зона ответственности |
| --- | --- | --- |
| Главный бухгалтер	| Контролирует корректность обработки документов и рассматривает документы, требующие ручной проверки. | Проверка проблемных документов, принятие решений по ошибкам, контроль корректности обработки, настройка правил поставщиков и счетов учета |

## Внешние участники процесса

| Участник	| Описание | Роль в процессе |
| --- | --- | --- |
| Контрагент | Внешняя организация или ИП, которые направляют УПД | Передает входящий документ через ЭДО или почту |
| Клиент | Компания для которой ведется бухгалтерский учет. | Предоставляет доступ к своей базе 1С, сервису ЭДО. |

## Системные участники

| Участник	| Описание | Роль в процессе |
| n8n | Система автоматизации, которая управляет обработкой входящих документов | Получает документы, извлекает данные, выполняет проверки, формирует статус обработки, отправляет уведомления и передает данные в 1С |
| Административная панель | Интерфейс на базе n8n webhooks для управления правилами поставщиков  | Позволяет главному бухгалтеру создавать, изменять, включать и выключать правила счетов учета поставщиков |

## Внешние информационные системы

| Система | Назначение | Использование в процессе |
| --- | --- | --- |
| СБИС | Система ЭДО | Используется как источник входящих документов, получаемых через API |
| Диадок | Система ЭДО | Используется как источник входящих документов, перенаправляемых на почту |
| Почта | IMAP / SMTP | Используется для получения входящих документов, отправка уведомлений главному бухгалтеру |
| OCR / AI API | Внешний сервис распознования документов | Используется для извлечения данных из PDF и изображений |
| DaData | Внешний сервис проверки контрагентов | Проверка контрагентов по ИНН и КПП в реестре ФНС |
| 1C Fresh | Учетная система клиента | Поиск контрагентов, договоров, документов, номенклатуры, создание итогового документа |
| n8n Data Tables | Внутреннее  хранилище n8n | Хранение правил поставщиков |

## Основные сценарии взаимодействия
### Передача документа:
Контрагент направляет УПД через СБИС или Диадок, почту.
Если документ поступает через СБИС, n8n получает его череp API. Если документ поступает из Диадок. Он предварительно направляется на электронную почту и далее обрабатывается как почтовое вложение.

### Автоматическая обработка документов
n8n получает документ, определяет тип файла, извлекает данные, приводит их к единому JSON формату и выполняет проверки.
Для проверок n8n обращается к внешним сервисам: DaData и 1С Fresh

### Успешная обработка
Если документ проходит проверки, n8n передает данные в 1С Fresh и создает документ "поступления товаров/услуг"

### Ручная проверка
Если документ не проходит хотя бы одну проверку, n8n присваивает ему статус "Требует ручной проверки" и отправляет уведомление гл. бухгалтеру
Гл. бухгалтер анализирует ошибки и принимает решение по дальнейшей обработке документа. 

### Управление правилами поставщиков
Гл. Бухгалтер через административынй интерфейс управляет правилами поставшиков и счетами учета. 
Правила сохраняются в Data Tables и используются при обработке документов.
