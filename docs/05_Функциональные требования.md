# Функциональные требования

| ID | Требование |
| --- | --- |
| FR-001| Система должна принимать входящие документы из СБИС по расписанию: каждые 15 минут проверять все новые документы за последние 3 часа |
| FR-002| Система должна принимать входящие письма и вложенные документы с почты |
| FR-003| Система должна определять тип входящего документа: XML, PDF, Изображение или неизвестный формат |
| FR-004| Система должна защищать от повторной обработки документы из СБИС, если документ с таким же индентификатором СБИС уже был обработан |
| FR-005| Система должна извлекать реквизиты УПД из XML |
| FR-006| Система должна извлекать данные из PDF или изображений через OCR-обработку |
| FR-007| Система должна извлекать реквизиты документа в единый структурированный JSON-формат для последующей проверки |
| FR-008| Система должна проверять наличие дубля документа в 1с по комбинации полей : номер документа, дата, контрагент |
| FR-009| Система должна проверять математику НДС по формуле Сумма без НДС = Сумма с НДС − НДС. | 
| FR-010| Система должна проверять корректность и актуальность  ИНН и КПП контрагента в реестре ФНС |
| FR-011| Система должна проверять наличие ИНН и КПП контрагента в справочнике "Контрагенты"  1с |
| FR-012| Система должна проверять наличие ИНН и КПП покупателя  в справочнике "Организации" в  1с |
| FR-013| Система должна проверять входящий документ соответсвие договору в 1с, по полям: номер и дата договора |
| FR-014| Система должна определять тип документа: товар, материал, лизинг, услуга. |
| FR-015| Для позиций типа "Товар" система должна сопостовялть позицию со справочником номенклатуры 1с по полям: наименование, ОКПД2, единица измерения |
| FR-016| Для позиций типа "Услуга" система должна |
| FR-017| Для позиций типа "Лизинг" система должна |
| FR-018| Для позиций типа "Материал" система должна |
| FR-019| |
| FR-020| |
| FR-021| |
| FR-022| Если документ не прошел одну или несколько проверок, система должна присвоить статус "Требует ручной проверки" и сохранить список причин |
| FR-023| Система должна уведомлять бухгалетра по почте, если документ в статусе "Требует ручной проверки"|
| FR-023| Почтовое уведомление бухгалтеру должно содержать реквизиты документа, описание причин непройденых проверок |
