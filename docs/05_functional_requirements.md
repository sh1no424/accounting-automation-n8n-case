# Функциональные требования

| ID | Требование |
| --------- | --- |
| FR-001 | Система должна принимать входящие документы из СБИС по расписанию. |
| FR-002 | Система должна принимать входящие письма и вложенные документы с почты |
| FR-003 | Система должна определять тип входящего документа: XML, PDF, Изображение или неизвестный формат |
| FR-004 | Система должна защищать от повторной обработки документы из СБИС, если документ с таким же id СБИС уже был обработан |
| FR-005 | Система должна извлекать реквизиты УПД из XML |
| FR-006 | Система должна извлекать данные из PDF или изображений. |
| FR-007 | Система должна приводить реквизиты документа в единый структурированный JSON-формат для последующей проверки |
| FR-008 | Система должна проверять наличие дубля документа в 1с по комбинации полей : номер документа, дата, контрагент |
| FR-009 | Система должна проверять корректность расчета НДС. | 
| FR-010 | Система должна проверять корректность и актуальность  ИНН и КПП контрагента в реестре ФНС |
| FR-011 | Система должна проверять наличие ИНН и КПП контрагента в справочнике "Контрагенты"  1с |
| FR-012 | Система должна проверять наличие ИНН и КПП покупателя  в справочнике "Организации" в  1с |
| FR-013 | Система должна проверять входящий документ соответсвие договору в 1с, по полям: номер и дата договора |
| FR-014 | Система должна определять тип документа: товар, материал, лизинг, услуга. |
| FR-015 | Для позиций типа "Товар" система должна сопоставлять позицию со справочником номенклатуры 1с по полям: наименование, ОКПД2, единица измерения |
| FR-016 | Для позиций типа "Услуга" система должна сопоставлять позицию со справочником номенклатуры 1с по наименованию |
| FR-017 | Если услуга не найдена в номенклатуре 1с, система должна подставлять значение "Услуги" в шапку позиции документа. | 
| FR-018 | Система должна сохранять наименование услуги в содержание документа | 
| FR-019 | Для позиций типа "Лизинг" система должна подставлять шапка - "лизинг", а наименование в содеражание. |
| FR-020 | Для позиций типа "Материал" система должна сопоставлять позицию со справочником номенклатуры 1с по полям: наименование, ОКПД2, единица измерения|
| FR-021 | Если материал не найден в справочнике номенклатуры 1с, система должна создавать новую позицию номенклатуры. |
| FR-022 | Система должна проверять корректность заполнения строк 5a и 5b УПД |
| FR-023 | Если документ прошел проверку, система должна присваивать статус "Принят к учету"  |
| FR-024 | Система должна создавать документ поступления на основе входящего документа, если документ в статусе "Принят к учету" |
| FR-025 | Если документ не прошел одну или несколько проверок, система должна присвоить статус "Требует ручной проверки" и сохранить список причин |
| FR-026 | Система должна уведомлять бухгалтера по почте, если документ в статусе "Требует ручной проверки"|
| FR-027 | Почтовое уведомление бухгалтеру должно содержать реквизиты документа, описание причин непройденных проверок |
| FR-028 | Система должна иметь интерфейс для управления правилами счетов учета для поставщиков |
| FR-029 | Пользователь должен иметь возможность создавать правило счетов учета |
| FR-030 | Пользователь должен иметь возможность изменять правило счетов учета |
| FR-031 | Пользователь должен иметь возможность отключать и включать правила счетов учета |
| FR-032 | Интерфейс управления счетов учета должен предоставлять возможность добавлять правила для счета расчетов, счета авансов, счета учета, счета НДС для конкретного контрагента по ИНН и КПП |
| FR-033 | Система должна сохранять правила поставщиков в Data Table n8n |
| FR-034 | Система должна проверять корректность ИНН и КПП при сохранении правила |
| --------- | --- |
