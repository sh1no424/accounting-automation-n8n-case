# Проект: Автоматизация документооборота в 1с через n8n. Прием и обработка УПД.

## Описание
Проект представляет собой автоматизацию приема первичных документов с использованием n8n
- Основной пользователь процесса - главный бухгалтер, который контролирует выполнение процесса, решает спорные моменты
- Решение на базе n8n автоматически получает документы из СБИС и почты, извлекает данные XML, PDF, Сканов, проверяет реквизиты, соответствие договору, номенклатуру, схождение сумм НДС, заполнение строк 5a/5б. создает документ «Поступление товаров/услуг» на основании УПД в 1с, подставляет контрагента по ИНН, выбирает договор по номеру/наименованию, заполняет табличную часть, автоматически контролируя тип документа, отражение по счетам, дублирование документов по номенклатуре, по одному и тому же договору/периоду.
- Дополнительно была реализована админка для управления правилами поставщиков и счетами учета. 
- Автоматизация позволяет, по предварительной оценке, сократить до 20% рабочего времени бухгалтера. 

## Моя роль в проекте

 - Анализ бизнес-процесса обработки первичных документов
 - Выделение бизнес-правил и контрольных точек
 - Проектирование workflow в n8n
 - Настройка HTTP API-интеграций
 - Работа с OData API 1C
 - Обработка XML и JSON данных
 - Написание и адаптация JS-обработок
 - Проектирование статусов и обработки ошибок
 - Тестирование сценариев обработки.

## Бизнес-проблема

До автоматизации обработка первичных документов выполнялась медленно, с большим количеством ручных действий. Бухгалтеру необходимо было вручную проверять реквизиты, контролировать дубли, сопоставлять документы с договорами и номенклатурой, проверять суммы НДС, создавать документы в 1с.

Основные проблемы:
 - Низкая скорость обработки.
 - Риски дублирования документа
 - Риски ошибок в заполнении реквизитов, договоров, номенклатуры
 - Ошибки в расчетах сумм и НДС

 ## Цель проекта

 Цель проекта - автоматизировать прием, проверку и обработку входящих УПД, чтобы снизить нагрузку на бухгалтера, уменьшить риск ошибок и повысить прозрачность контроля документов.

 Система должна обеспечивать:

  - Автоматический прием входящих документов из СБИС или почты;
  - Нормализация входящих документов в единый  JSON формат;
  - Проверка ИНН и КПП контрагента в реестре ФНС;
  - Поиск контрагента, договора и номенклатуры в 1С;
  - Выявление дублей документов;
  - Проверку сумм НДС
  - проверку строк 5a/5b;
  - Создание документа "поступление товаров/услуг" в 1С;
  - Уведомление главного бухгалтера о документах, требующих ручной проверки.

## Основной процесс

1. Документ поступает из СБИС или на почту Gmail/
2. Workflow определяет тип документа: XML, PDF, изображение или неизвестный формат.
3. XML документ парсится и декодируется.
4. PDF или изображение отправляется на OCR-обработку. 
5. Извлеченные данные приводятся к единому json-формату. 
6. Документ в json-формате проходит проверки:
    * Проверяется математика НДС
    * Проверяется ИНН контрагента в реестре ФНС
    * Проверяется наличие контрагента в 1с
    * Проверяется наличие договора в 1с 
    * Проверяется дублированние документа
    * Проверяются строки 5a\5b
    * Проверяется наличие правил для счета учета
    * Номенклатура классифицируется: Услуга, Лизинг, Материалы, Товары
7. На основе проверок формируется статус документа: принят к учету или требует ручной проверки. 
8. Если ручная проверка не требуется, документ создается в 1с.
9. Если требуется ручная проверка, главному бухгалтеру отправляется уведомление с содержанием документа и проблемами в нем.
