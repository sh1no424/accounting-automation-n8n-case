# Проект: Автоматизация документооборота в 1с через n8n. Прием и обработка УПД.

## Описание

Кейс автоматизации бухгалстерского документооборота

## Цель проекта 

Проект направлен на автоматизацию обработки первичных документов, снижение количества ошибок и повышения прозрачности их контроля 

## Моя роль в проекте

 - Анализ процесса
 - Выделение бизнес правил и контрольных точек
 - Проектирование workflow в n8n
 - Настройка HTTP API-интеграций
 - Работа с OData 1C
 - Обработка JSON данных
 - Написание и адаптация JS-обработок
 - Проектирование статусов и обработки ошибок
 - Тестирование

 ## Стек технологий

  - n8n
  - HTTP Request
  - OData 1C
  - API СБИС
  - SMTP Gmail
  - JavaScript
  - JSON
  - LLM tools
  - XML

## Основной процесс

Основной пользователь процесса - главный бухгалтер, который контролирует выполнение процесса, решает спорные моменты. 
До автоматизации главными проблемами были: скорость обработки, риск ошибок, дублирование документов.
Решение на базе n8n автоматически проверяет реквизиты, соответствие договору, номенклатуру, схождение сумм НДС, заполнение строк 5a/5б. создает документ «Поступление товаров/услуг» на основании УПД в 1с, подставляет контрагента по ИНН, выбирает договор по номеру/наименованию, заполняет табличную часть, автоматически контролируя тип документа, отражение по счетам, дублирование документов по номенклатуре, по одному и тому же договору/периоду.
В процессе используются интеграции с Odata 1c, Aitunnel API, СБИС API, DaData API, Google Gmail SMTP-протокол
Результатом является автоматизированный процесс приема документов, с почты и СБИС, который освобождает 20% рабочего времени бухгалтера.
